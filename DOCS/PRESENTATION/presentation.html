Aim & Objectives

To develop a comprehensive, multi-functional Agriculture Support System that empowers farmers by providing real-time information, service access, early warnings, and easy access to schemes and machinery through a user-friendly interface that supports regional languages and voice inputs.

🔄 Real-time Information

Providing instant, relevant data to farmers including weather, risk alerts, and advisories.

🛒 Enhanced Service Access

Connecting farmers with machinery owners through a rental marketplace with search and booking.

⚠️ Early Warning Systems

Alerting farmers to potential threats like disease outbreaks through AI-powered detection.

📋 Access to Resources

Facilitating easier access to government schemes and agricultural machinery.

Challenges Faced and Solutions Implemented
  • Challenge: User registration was failing with 500 error and frontend JSON parse errors. Solution: Fixed JWT payload serialization (`ObjectId` to string), added robust backend error responses, and improved frontend response parsing (`REGISTRATION_FIX.md`).
  • Challenge: Crop disease analysis broke due incompatible Gemini SDK usage. Solution: Migrated to `@google/genai` and rewrote `src/lib/gemini.ts` with schema-based structured output handling (`CROP_DISEASE_FIX.md`).
  • Challenge: Securing AI API keys from frontend exposure. Solution: Implemented backend proxy endpoints (`/api/ai/*`) with server `.env` keys and optional user keys, removing direct browser key exposure (`API_KEY_SECURITY.md`).
  • Challenge: Service worker caching caused stale/blank pages during development refresh. Solution: Switched to network-first caching and added smart registration to disable SW in dev and clear old caches (`SERVICE_WORKER_FIX.md`).
  • Challenge: End-to-end machinery workflow had partial booking-status and messaging integration. Solution: Verified core booking routes, documented partial endpoints, and moved full lifecycle closure to Semester 8 with integration testing (`API_INDEX.md`, `VERIFIED_ROUTES_QUICK_REF.txt`).
  • Challenge: Voice recognition quality varied by accent/noise conditions. Solution: Added language-specific configurations, fallback text input, and planned deeper multilingual validation in Semester 8.
